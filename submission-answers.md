# 질문 5문항 답변 초안

## 1. 무엇을, 누가, 어떤 상황에서 쓰나요?

삼일PwC 페르소나의 감사·시장분석·AX 컨설팅 탐색을 돕는 3모듈 Codex 플러그인입니다. 감사·영업·리스크 담당자는 상장사의 최근 완료연도 감사인, 감사인 이력, 데이터 공백과 감사보수·시간을 확인하고, 연도별 공개시장 점유율을 비교할 수 있습니다. AX 컨설턴트는 고객이 형식 없는 말로 설명한 업무를 한 단계씩 인터뷰해 현재 프로세스, 예외, 통제, 자동화 후보와 사람 승인 경계가 담긴 기준본으로 만들 수 있습니다. 공개 데모는 삼일 내부 데이터나 고객 기밀을 사용하지 않으며, 실제 수임 판단에는 독립성·품질관리·전문가 검토가 별도로 필요합니다.

## 2. 왜 이 문제를 선택했나요?

회계 AX에는 회계·감사 도메인 지식과 고객 업무의 실제 흐름을 함께 이해하는 과정이 필요하지만, 공개 감사정보는 여러 공시와 표에 흩어져 있고 고객 업무기술서의 형식도 제각각입니다. 그래서 공개 근거로 감사시장과 기업을 탐색하는 기능, 시장 전체를 비교하는 기능, 그리고 고객 설명을 표준화된 AX 기회지도로 바꾸는 기능을 하나의 제품 흐름으로 묶었습니다. 이를 통해 “어느 기업을 볼 것인가”에서 “어떤 업무를 어떻게 바꿀 것인가”까지 이어지는 AX 컨설팅의 초기 탐색 비용을 줄이는 것이 목표입니다.

## 3. 플러그인은 어떻게 작동하나요?

첫째, `audit-radar`는 OpenDART에서 상장사를 검색해 요청 연도의 사업보고서, 감사인·감사의견, 감사보수·시간과 최근 공시 공백을 한 화면에 정리합니다. 구조화 데이터가 비어 있을 때는 같은 사업연도의 감사의견표와 감사용역표가 일치하는 경우에만 원문 XML을 보완 근거로 사용하며, 감사인 이력만으로 미래 선임기간이나 주기적 지정 D-day를 추정하지 않습니다.

둘째, `audit-market-share`는 2026-07-23 현재 확보 가능한 2023~2025 사업보고서 제출회사 9,900행을 회계법인 그룹별로 재집계합니다. 회사 수, 계약보수, 실제보수, 피감사회사 공시 매출·수익을 각각 독립된 분모와 커버리지로 보여주며, 원문 단위 충돌과 외화 보수는 접수번호가 남는 검증 예외 또는 보수적 제외로 처리합니다.

셋째, `map-workflow-ax`는 업무기술서 양식을 요구하지 않고 사용자와 한 질문씩 대화합니다. 시작·종료, 정상 흐름, 분기·예외, 시스템, 데이터와 사람 승인 지점을 확인한 뒤 사용자가 전체 흐름을 확정해야 AX 후보를 생성합니다. 구현할 후보를 고르면 기준본 해시와 완료 조건이 담긴 인계서를 만들고, 실제 툴 제작은 별도 Codex 작업에서 진행하도록 분리합니다.

## 4. AI를 어떻게 활용했나요?

AI와 Codex를 공개자료 조사, 원문 표 구조 분석, 단위·기간·외화 정규화 규칙 설계, 회귀테스트 작성과 웹 구현에 활용했습니다. Process-to-AX에서는 AI가 비정형 업무 설명을 단계·분기·예외·통제로 구조화하고 자동화 후보를 제안하지만, 확인되지 않은 처리량이나 효과를 만들지 않으며 사용자 확정 전에는 분석을 잠급니다. 감사인 선임, 공시 단위 충돌, AX 우선순위와 최종 통제 판단은 사람이 검토하도록 경계를 명시했습니다.

## 5. 어떻게 검증했나요?

감사인 레이더는 삼성전자, SK하이닉스, 현대자동차, KB금융, NAVER, POSCO홀딩스, JTC 등 12개사를 회사 IR·법령·DART 원문과 교차검사했습니다. 그 과정에서 미래 3년 자동 연장, 비12월 결산 최신연도 누락, 전기 행의 당기 오인식, 정정공시 중복과 공시 제출인명 오용을 제거했습니다. 점유율 데이터는 2023년 3,234개사, 2024년 3,323개사, 2025년 3,343개사의 중복키·접수번호·음수·비유한값을 검사하고, 의심 단위와 외화는 원문 접수번호로 재대사했습니다. Process-to-AX는 세션 상태, 스키마, 평가식, 별도 구현 인계와 재개 흐름을 합성 시나리오로 검사했습니다. 마지막으로 공개 Render 주소에서 세 화면, CSV 로딩, 연도 전체 툴팁과 대표 회사 검색을 다시 확인합니다.
