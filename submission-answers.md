# 질문 5문항 답변 초안

## 1. 무엇을, 누가, 어떤 상황에서 쓰나요?

회계법인 ERP/CRM에 통합되는 감사영업 대상 추천을 보조하는 Codex 플러그인입니다. 회계법인 감사·영업·리스크 담당자가 특정 기업 또는 검색 후보군을 설정된 firm context로 평가해, 현재 감사인, 최근 감사인 이력, 동일 감사인 연속연차, 주기적 지정 또는 자유선임 전환 가능성, 영업 적합도 점수, 추천등급, 첫 컨택 각도를 빠르게 확인할 때 사용합니다. 상장사뿐 아니라 OpenDART에 외부감사관련 감사보고서가 공시된 비상장 외감대상 회사도 보조적으로 탐지하고, 상장사·대형비상장/사업보고서 제출대상 후보·비상장 외감대상·금융회사 추정·유한회사 추정·외감요건 후보처럼 영업 검토 케이스를 나눕니다. 실제 서비스 맥락에서는 회계법인 측 인력의 업종 감사경험·도메인 지식, 감사대상 회사 의사결정권자의 role/tag, 학력·이력·네트워크 기반 관계 edge, 매출추이, 감사지출비용 추이까지 함께 고려합니다.

## 2. 왜 이 문제를 선택했나요?

회계법인의 본질적 업무는 외부감사이며, 감사용역은 기업과 장기적 신뢰 네트워크를 만드는 출발점입니다. 주기적 지정제에 따라 상장회사 등은 6년 자유선임 후 3년 지정감사로 전환될 수 있어, 회사와 회계법인 모두 감사인 변경 이벤트를 미리 파악해야 합니다. OpenDART에는 사업보고서의 감사인·감사의견 정보와 외부감사관련 감사보고서 공시목록이 공개되어 있으므로 공개 자료로 검증 가능한 문제입니다. 단순 정보 조회에 그치지 않고, 실제 서비스에서는 회계법인 ERP/CRM의 기존 관계, 제한 계정, 담당 산업, 서비스 라인, 내부 인력 역량, 의사결정권자 접점 신호와 결합해 영업 우선순위를 추천하는 방식으로 확장할 수 있습니다.

## 3. 플러그인은 어떻게 작동하나요?

플러그인은 `audit-radar` 스킬과 `scripts/audit_radar.py` 실행 스크립트로 구성됩니다. OpenDART 고유번호 목록으로 기업을 검색하고, `회계감사인의 명칭 및 감사의견` API를 여러 사업연도에 걸쳐 조회합니다. API에서 빠지는 비상장 외감대상 가능성을 보완하기 위해 OpenDART 공시검색의 `외부감사관련` 감사보고서·연결감사보고서 목록도 함께 조회하고, 공시 제출인을 감사인 후보로 병합합니다. 감사인 명칭을 정규화해 동일 감사인 연속연차를 계산하고, 6년 자유선임·3년 지정감사 구조를 바탕으로 다음 지정/선임 이벤트를 추정합니다. 이후 `firm_context.sample.json` 또는 사용자가 제공한 `firm_context.local.json`을 기준으로 타이밍, 세그먼트 적합도, 공개 검증성, 부가자문 확장성, ERP/CRM 맥락, 인력·관계 커버리지, 제약 조정을 점수화해 추천등급과 첫 컨택 각도를 제안합니다. 인력·관계 커버리지는 업종 감사경험 보유 인력, 감사대상 회사 의사결정권자 role/tag, 학력·이력·네트워크 태그 교집합, 매출추이와 감사지출비용 추이를 집계 신호로만 사용합니다. `감사전재무제표미제출신고서`, 제출 지연/연장, 정정 공시는 특이사항으로 따로 표시합니다.

## 4. AI를 어떻게 활용했나요?

AI를 사용해 과제 요구사항을 해석하고, 회계법인의 감사영업 맥락, OpenDART 공개 API, 외부감사인 주기적 지정 제도에 대한 공개 자료를 조사했습니다. 이후 문제 범위를 좁혀 감사인 매칭·연속연차·다음 이벤트 추정 기능으로 설계하고, 단순 조회 서비스가 아니라 ERP/CRM firm context와 결합된 추천 시스템으로 제품 방향을 재정의했습니다. Codex 플러그인 manifest, 스킬 문서, Python 기반 OpenDART 연동 코드, 웹 UI, README와 답변 초안을 AI와 협업해 작성했습니다.

## 5. 어떻게 검증했나요?

API 키 없이도 동작 확인이 가능한 데모 감사인 이력과 샘플 firm context로 연속연차, 지정감사 가능성 추정, 추천등급 산정 로직을 검증합니다. 외부감사 공시목록 샘플로 감사보고서 기간 추출, 공시 제출인 기반 감사인 변환, 미제출·정정 특이사항 분리 로직을 검증했습니다. 영업 케이스 분류는 상장사, 정기보고서 API 확인 회사, 외부감사 감사보고서 공시 확인 회사, 금융·유한회사 키워드 후보를 나눠 표시하는지 확인합니다. 샘플 firm context에는 실명 개인정보가 아닌 역할·태그·관계 edge만 넣어 인력·관계 커버리지 점수 근거가 생성되는지 확인합니다. OpenDART API 키를 설정한 환경에서는 실제 기업명 검색과 추천 후보 랭킹이 동작하는지 확인했습니다. Render 공개 배포 URL(`https://samil-audit-radar.onrender.com`)에서 `/healthz`, `/api/status`, 삼성전자 검색 API 응답, 웹 UI 렌더링을 확인했습니다. 또한 plugin validator로 `src/.codex-plugin/plugin.json`과 스킬 구조가 Codex 플러그인 형식에 맞는지 검증합니다.
