# 질문 5문항 답변 초안

## 1. 무엇을, 누가, 어떤 상황에서 쓰나요?

삼일PwC의 감사 시장 리서치와 감사인 선임/지정 타이밍 모니터링을 보조하는 Codex 플러그인입니다. 회계법인 감사·영업·리스크 담당자가 특정 기업의 현재 감사인, 최근 감사인 이력, 동일 감사인 연속연차, 주기적 지정 또는 자유선임 전환 가능성을 빠르게 확인할 때 사용합니다. 상장사뿐 아니라 OpenDART에 외부감사관련 감사보고서가 공시된 비상장 외감대상 회사도 보조적으로 탐지합니다.

## 2. 왜 이 문제를 선택했나요?

삼일PwC의 본질적 업무는 외부감사이며, 감사용역은 기업과 장기적 신뢰 네트워크를 만드는 출발점입니다. 주기적 지정제에 따라 상장회사 등은 6년 자유선임 후 3년 지정감사로 전환될 수 있어, 회사와 회계법인 모두 감사인 변경 이벤트를 미리 파악해야 합니다. OpenDART에는 사업보고서의 감사인·감사의견 정보와 외부감사관련 감사보고서 공시목록이 공개되어 있으므로 공개 자료로 검증 가능한 문제입니다.

## 3. 플러그인은 어떻게 작동하나요?

플러그인은 `audit-radar` 스킬과 `scripts/audit_radar.py` 실행 스크립트로 구성됩니다. OpenDART 고유번호 목록으로 기업을 검색하고, `회계감사인의 명칭 및 감사의견` API를 여러 사업연도에 걸쳐 조회합니다. API에서 빠지는 비상장 외감대상 가능성을 보완하기 위해 OpenDART 공시검색의 `외부감사관련` 감사보고서·연결감사보고서 목록도 함께 조회하고, 공시 제출인을 감사인 후보로 병합합니다. 감사인 명칭을 정규화해 동일 감사인 연속연차를 계산하고, 6년 자유선임·3년 지정감사 구조를 바탕으로 다음 지정/선임 이벤트를 추정합니다. `감사전재무제표미제출신고서`, 제출 지연/연장, 정정 공시는 특이사항으로 따로 표시합니다.

## 4. AI를 어떻게 활용했나요?

AI를 사용해 과제 요구사항을 해석하고, 삼일PwC의 본업과 OpenDART 공개 API, 외부감사인 주기적 지정 제도에 대한 공개 자료를 조사했습니다. 이후 문제 범위를 좁혀 감사인 매칭·연속연차·다음 이벤트 추정 기능으로 설계하고, Codex 플러그인 manifest, 스킬 문서, Python 기반 OpenDART 연동 코드, 로컬 웹 UI, README와 답변 초안을 AI와 협업해 작성했습니다.

## 5. 어떻게 검증했나요?

API 키 없이도 동작 확인이 가능한 데모 감사인 이력으로 연속연차와 지정감사 가능성 추정 로직을 검증합니다. 외부감사 공시목록 샘플로 감사보고서 기간 추출, 공시 제출인 기반 감사인 변환, 미제출·정정 특이사항 분리 로직을 검증했습니다. OpenDART API 키를 설정한 환경에서는 실제 기업명 검색이 동작하는지 확인했습니다. Render 공개 배포 URL(`https://samil-audit-radar.onrender.com`)에서 `/healthz`, `/api/status`, 삼성전자 검색 API 응답, 웹 UI 렌더링을 확인했습니다. 또한 plugin validator로 `src/.codex-plugin/plugin.json`과 스킬 구조가 Codex 플러그인 형식에 맞는지 검증합니다.
