# 질문 5문항 답변 초안

## 1. 무엇을, 누가, 어떤 상황에서 쓰나요?

삼일PwC 페르소나의 상장사 감사영업 대상 추천을 보조하는 Codex 플러그인입니다. 회계법인 감사·영업·리스크 담당자가 상장사를 검색해 현재 감사인, 최근 감사인 이력, 동일 감사인 연속연차, 주기적 지정 또는 자유선임 전환 가능성, 감사용역 보수·시간, 임원 주요경력, 영업 적합도 점수, 추천등급, 첫 컨택 각도를 빠르게 확인할 때 사용합니다. 실제 서비스 맥락에서는 회계법인 측 인력의 업종 감사경험·도메인 지식, 독립성 제한, 기존 관계, 감사위원회·감사·사외이사 후보군과의 합법적 접점 신호를 함께 고려할 수 있습니다.

## 2. 왜 이 문제를 선택했나요?

회계법인의 본질적 업무는 외부감사이며, 감사용역은 기업과 장기적 신뢰 네트워크를 만드는 출발점입니다. 주기적 지정제에 따라 상장회사 등은 6년 자유선임 후 3년 지정감사로 전환될 수 있어, 회사와 회계법인 모두 감사인 변경 이벤트를 미리 파악해야 합니다. 상장사는 OpenDART 정기보고서 주요정보 API에서 감사인·감사의견, 감사용역 보수·시간, 임원 현황을 구조화해 확인할 수 있어 공개 자료 기반 검증 가능성이 높습니다. 단순 정보 조회에 그치지 않고, 삼일PwC의 감사·Tax·Deals·산업 전문성 맥락과 결합해 영업 우선순위를 추천하는 방식으로 설계했습니다.

## 3. 플러그인은 어떻게 작동하나요?

플러그인은 `audit-radar` 스킬과 `scripts/audit_radar.py` 실행 스크립트로 구성됩니다. OpenDART 고유번호 목록에서 종목코드가 있는 상장사 후보만 검색하고, `회계감사인의 명칭 및 감사의견` API를 여러 사업연도에 걸쳐 조회합니다. 감사인 명칭을 정규화해 동일 감사인 연속연차를 계산하고, 6년 자유선임·3년 지정감사 구조를 바탕으로 다음 지정/선임 이벤트를 추정합니다. `감사용역체결현황` API로 감사보수·감사시간을, `임원 현황` API로 대표이사·사외이사·감사위원회 관련 주요경력을 함께 표시합니다. 이후 삼일PwC firm context를 기준으로 타이밍, 세그먼트 적합도, 공개 검증성, 부가자문 확장성, ERP/CRM 확장 가능성, 인력·관계 커버리지, 제약 조정을 점수화해 추천등급과 첫 컨택 각도를 제안합니다. `감사전재무제표미제출신고서`, 제출 지연/연장, 정정 공시는 특이사항으로 따로 표시합니다. 임원 학력은 OpenDART 구조화 필드가 아니므로 주요경력 텍스트 안에 명시된 경우에만 단서로 봅니다.

## 4. AI를 어떻게 활용했나요?

AI를 사용해 과제 요구사항을 해석하고, 회계법인의 감사영업 맥락, OpenDART 공개 API, 외부감사인 주기적 지정 제도에 대한 공개 자료를 조사했습니다. 이후 문제 범위를 상장사로 좁혀 감사인 매칭·연속연차·감사보수·임원 주요경력·다음 이벤트 추정 기능으로 설계하고, 단순 조회 서비스가 아니라 삼일PwC 페르소나의 감사영업 추천 시스템으로 제품 방향을 재정의했습니다. Codex 플러그인 manifest, 스킬 문서, Python 기반 OpenDART 연동 코드, 웹 UI, README와 답변 초안을 AI와 협업해 작성했습니다.

## 5. 어떻게 검증했나요?

API 키 없이도 동작 확인이 가능한 데모 감사인 이력, 임원 주요경력 샘플, 삼일PwC firm context로 연속연차, 지정감사 가능성 추정, 추천등급 산정 로직을 검증합니다. 영업 케이스 분류는 상장사, 금융회사 키워드 후보, 감사인 교체 타이밍 후보, 감사위원회 후보군 신호를 표시하는지 확인합니다. 공개 데모에는 실명 개인정보나 삼일 내부 데이터가 없고, 임원 학력은 구조화 필드가 아니라 주요경력 텍스트 단서로만 다룹니다. OpenDART API 키를 설정한 환경에서는 실제 기업명 검색, 삼성전자 리포트, 감사용역 보수·시간, 임원 현황, 추천 후보 랭킹이 동작하는지 확인했습니다. Render 공개 배포 URL(`https://samil-audit-radar.onrender.com`)에서 `/healthz`, `/api/status`, 삼성전자 검색 API 응답, 웹 UI 렌더링을 확인했습니다. 또한 plugin validator로 `src/.codex-plugin/plugin.json`과 스킬 구조가 Codex 플러그인 형식에 맞는지 검증합니다.
