# 질문 5문항 답변 초안

## 1. 무엇을, 누가, 어떤 상황에서 쓰나요?

삼일PwC 페르소나의 상장사 서비스 기회 추천을 보조하는 Codex 플러그인입니다. 회계법인 감사·영업·리스크 담당자가 상장사를 검색해 현재 감사인, 최근 감사인 이력, 동일 감사인 연속연차, 감사인 교체/재선임 검토일, 주기적 지정 또는 자유선임 전환 가능성, 감사용역 보수·시간, 임원 주요경력을 확인하고, 사람인 채용공고 키워드로 Assurance/Risk, Tax, Deals, Capital Markets, Finance Transformation 수요 신호를 조기 탐지할 때 사용합니다. 실제 서비스 맥락에서는 회계법인 측 인력의 업종 감사경험·도메인 지식, 독립성 제한, 기존 관계, 감사위원회·감사·사외이사 후보군과의 합법적 접점 신호를 함께 고려할 수 있습니다.

## 2. 왜 이 문제를 선택했나요?

회계법인의 본질적 업무는 외부감사이지만, 실제 성장 기회는 감사, Tax, Deals, 내부통제, 재무시스템 자문 중 어느 경로가 더 적합한지 먼저 판단하는 데 있습니다. 공시는 확정 이벤트 확인에는 유용하지만 영업 타이밍으로는 늦을 수 있으므로, 채용공고의 내부회계·연결결산·이전가격·M&A·IPO·ERP 키워드를 선제 신호로 사용합니다. 이후 OpenDART로 현재 감사인, 감사보수, 임원·감사위원회 정보를 확인해 독립성 및 접근 경로를 검토하도록 설계했습니다.

## 3. 플러그인은 어떻게 작동하나요?

플러그인은 `audit-radar` 스킬과 `scripts/audit_radar.py` 실행 스크립트로 구성됩니다. OpenDART 고유번호 목록에서 종목코드가 있는 상장사 후보만 검색하고, `회계감사인의 명칭 및 감사의견` API를 여러 사업연도에 걸쳐 조회합니다. `감사용역체결현황` API로 감사보수·감사시간을, `임원 현황` API로 대표이사·사외이사·감사위원회 관련 주요경력을 함께 표시합니다. 결산월과 외부감사인 선임 검토 기한을 반영해 3개 사업연도 선임 구간 종료, 6년 자유선임 후 주기적 지정 후보, 우수기업 유예 시나리오를 시간순 D-day 타임라인으로 산출합니다. 추가로 `jobs` 명령은 사람인 Job Search API를 seed 키워드별로 호출해 관련 채용공고를 수집하고, 키워드 룰로 Assurance/Risk, Tax, Deals, Capital Markets, Finance Transformation 신호를 점수화합니다. 이후 삼일PwC firm context와 독립성 검토 필요성을 함께 표시합니다.

## 4. AI를 어떻게 활용했나요?

AI를 사용해 과제 요구사항을 해석하고, 회계법인의 감사영업 맥락, OpenDART 공개 API, 외부감사인 주기적 지정 제도에 대한 공개 자료를 조사했습니다. 이후 문제 범위를 상장사로 좁혀 감사인 매칭·연속연차·감사보수·임원 주요경력·다음 이벤트 추정 기능으로 설계하고, 단순 조회 서비스가 아니라 삼일PwC 페르소나의 감사영업 추천 시스템으로 제품 방향을 재정의했습니다. Codex 플러그인 manifest, 스킬 문서, Python 기반 OpenDART 연동 코드, 웹 UI, README와 답변 초안을 AI와 협업해 작성했습니다.

## 5. 어떻게 검증했나요?

API 키 없이도 동작 확인이 가능한 데모 감사인 이력, 임원 주요경력 샘플, 삼일PwC firm context로 연속연차, 지정감사 가능성 추정, 추천등급 산정 로직을 검증합니다. 영업 케이스 분류는 상장사, 금융회사 키워드 후보, 감사인 교체 타이밍 후보, 감사위원회 후보군 신호를 표시하는지 확인합니다. 공개 데모에는 실명 개인정보나 삼일 내부 데이터가 없고, 임원 학력은 구조화 필드가 아니라 주요경력 텍스트 단서로만 다룹니다. OpenDART API 키를 설정한 환경에서는 실제 기업명 검색, 삼성전자 리포트, 감사용역 보수·시간, 임원 현황, 추천 후보 랭킹이 동작하는지 확인했습니다. Render 공개 배포 URL(`https://samil-audit-radar.onrender.com`)에서 `/healthz`, `/api/status`, 삼성전자 검색 API 응답, 웹 UI 렌더링을 확인했습니다. 또한 plugin validator로 `src/.codex-plugin/plugin.json`과 스킬 구조가 Codex 플러그인 형식에 맞는지 검증합니다.
