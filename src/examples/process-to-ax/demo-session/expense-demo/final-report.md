## 1. 확정된 현재 업무

- 목적: 정당한 업무경비를 승인·회계처리·지급하고 증빙을 보존한다.
- 시작: 임직원이 출장 또는 업무 관련 비용을 지출한다.
- 종료: ERP 전표와 필요한 지급이 승인되고 증빙이 보관된다.
- 포함 범위: 국내 출장·업무경비
- 제외 범위: 해외출장
- 결과 사용자: 신청 임직원 · 재무운영팀

## 2. AX 분석 요약

경비 신청의 반복 입력과 재무 검토의 증빙 대조를 보조하는 얇은 전용 업무함을 우선 검증하고, 부서 승인·전표 승인·지급 승인은 사람에게 남긴다. 처리량과 API 제공 여부가 미확인이므로 효과와 연동 방식은 PoC 전에 측정한다.

### 기회 포트폴리오

| 우선순위 | ID | 대상 단계 | 해결 유형 | 제안 | 신뢰도 |
|---|---|---|---|---|---|
| now | AX01 | S01 | dedicated-app | 증빙 기반 경비 신청 업무함 | medium |
| now | AX02 | S03 | hybrid | 증빙·규정 검토 예외 큐 | medium |
| do-not-automate | AX03 | S04 | human-only | 전표·지급 최종 승인 유지 | high |

### AX01. 증빙 기반 경비 신청 업무함

- 관련 단계: S01
- 문제: 신청자가 영수증과 카드 내역의 내용을 그룹웨어에 반복 입력한다.
- 해결 유형: `dedicated-app`
- 근거: 문자 인식만으로 끝나지 않고 영수증별 초안, 출장 단위 묶음, 수정 상태, 제출 이력과 사용자 확인을 지속적으로 관리해야 한다.
- 자동화 경계: 시스템은 증빙에서 거래처·일자·금액을 추출하고 신청 초안을 만든다. 신청자가 원문과 초안을 확인·수정한 뒤 직접 제출하며, 시스템은 제출이나 승인을 임의 실행하지 않는다.
- 가치 가설: 반복 입력과 누락 보완을 줄여 신청자의 작성시간과 반려율을 낮출 가능성이 있다.
- 제품 기능: 영수증·카드내역을 출장 단위로 묶는 임시보관함 · 추출값과 원문을 나란히 보여주는 수정 화면 · 반복 사용하는 업무 목적·프로젝트의 재사용 · 작성·수정·제출 상태와 변경 이력
- 연동: 그룹웨어 신청서 · 법인카드 포털 또는 카드내역 파일
- 통제: 추출값마다 원문 증빙 위치를 표시한다. · 신청자가 제출 전 모든 필수값과 금액 합계를 확인한다. · 낮은 신뢰도 또는 중복 가능성은 자동 제출하지 않고 검토 대상으로 표시한다.
- 위험: 영수증 문자 인식 오류 · 다른 출장이나 프로젝트로 잘못 묶이는 오류 · 개인정보와 카드정보 노출
- 가정: 영수증 이미지 또는 카드내역 파일을 디지털로 입력할 수 있다.
- 미확인: 그룹웨어와 카드 포털의 API 제공 여부 · 영수증 이미지 품질
- 검증 지표: 신청서 1건당 작성시간 · 추출값 사용자 수정률 · 최초 제출 후 반려율

근거 문장:
- `S01` — 영수증을 모아서 그룹웨어에 하나씩 입력하고 팀장 결재를 올립니다.

### AX02. 증빙·규정 검토 예외 큐

- 관련 단계: S03
- 문제: 재무운영 담당자가 신청서와 증빙을 반복 대조하고 규정 위반 가능성을 수작업으로 찾는다.
- 해결 유형: `hybrid`
- 근거: 정확한 금액·일자 비교는 규칙으로 처리하고, 비정형 영수증 판독과 규정 설명은 AI가 보조하며, 예외의 최종 판단과 반려는 담당자가 수행해야 한다.
- 자동화 경계: 시스템은 증빙 필드와 신청값의 불일치, 누락, 규칙 후보를 표시하고 보완 요청 초안을 만든다. 담당자가 근거를 확인해 예외 인정, 반려 또는 통과를 결정한다.
- 가치 가설: 정상 건의 전수 확인보다 예외 중심 검토로 전환해 사람 검토시간과 재작업을 줄일 가능성이 있다.
- 제품 기능: 불일치·누락·낮은 신뢰도 건만 모으는 예외 업무함 · 신청값·증빙 원문·적용 규칙의 근거 연결 · 담당자 배정, 검토 의견, 보완 요청과 재제출 이력 · 반려 사유 태그와 반복 원인 대시보드
- 연동: 그룹웨어 승인 신청서 · 경비 규정 문서 · Excel 작업 파일
- 통제: 금액·일자·중복 검사는 결정적 규칙으로 재검증한다. · 규정 문구와 증빙 원문을 함께 표시하고 담당자가 처리 결과를 확정한다. · 규정 버전, 입력, 모델 결과, 사용자 수정과 최종 판단을 기록한다.
- 위험: 규정 해석 오류 · 정상 거래를 이상으로 표시하는 거짓 양성 · 보완 자료가 다른 채널에 남아 이력이 끊기는 문제
- 가정: 적용할 경비 규정의 유효 버전을 식별할 수 있다.
- 미확인: 월 처리량과 현재 검토시간 · 반려 사유별 실제 빈도
- 검증 지표: 신청서 1건당 재무 검토시간 · 예외 탐지 정밀도와 재현율 · 보완 요청 후 재처리시간

근거 문장:
- `S03` — 재무팀에서 엑셀로 내려받아 영수증이랑 금액을 맞춰보고 규정에 맞는지도 확인합니다.

### AX03. 전표·지급 최종 승인 유지

- 관련 단계: S04
- 문제: 전표와 자금 실행은 재무적 영향이 크고 권한 있는 승인자의 책임을 유지해야 한다.
- 해결 유형: `human-only`
- 근거: 사용자가 최종 지급 승인 자동화를 명시적으로 금지했고, 전표·이체 승인은 비가역적 외부 실행과 책임을 수반한다.
- 자동화 경계: 시스템은 승인자가 확인할 합계, 변경 내역, 예외와 근거를 준비할 수 있지만 전표 승인과 은행 이체 승인을 실행하지 않는다.
- 가치 가설: 사람의 최종 책임을 유지하면서 앞 단계의 정확한 근거 제공으로 승인 품질을 보조할 수 있다.
- 제품 기능: -
- 연동: -
- 통제: 권한 있는 승인자가 ERP와 인터넷뱅킹에서 각각 최종 승인한다. · 신청·검토·승인·지급 역할과 시각을 감사 이력으로 보존한다.
- 위험: 잘못된 전표 승인 · 오지급 또는 중복 지급 · 권한 분리 훼손
- 가정: -
- 미확인: ERP 업로드 준비 단계에서 허용되는 자동화 범위
- 검증 지표: 승인 전 발견된 오류 건수 · 권한 분리 위반 건수

근거 문장:
- `S04` — 재무팀이 ERP 업로드용 엑셀을 만들고 전표를 올리며 지급 승인은 자동화하면 안 될 것 같습니다.

## 3. MVP 제안

- 선택 기회: AX01 · AX02
- 포함 범위: 영수증 파일 업로드와 필드 추출 · 신청 초안의 사용자 확인·수정 · 신청값과 증빙의 금액·일자 대조 · 불일치·누락 예외 큐와 검토 이력
- 제외 범위: 업무 관련성 자동 승인 · 회계·세무 예외의 최종 판단 · ERP 전표 자동 승인 · 은행 지급 자동 승인
- 사용자 스토리: 신청자로서 영수증의 같은 내용을 반복 입력하지 않고 초안을 확인하고 싶다. · 재무운영 담당자로서 모든 신청이 아니라 불일치와 누락이 있는 건부터 검토하고 싶다.
- 사람 승인 지점: 신청자의 경비 신청 제출 · 부서 승인자의 업무 관련성 승인 · 재무운영 담당자의 예외 처리 · ERP 전표 및 은행 지급 최종 승인
- 성공 지표: PoC 전후 신청서 작성시간 · PoC 전후 재무 검토시간 · 추출값 사용자 수정률 · 최초 제출 후 반려율
